## Pull request guidelines

When you're ready to submit a pull request, keep the following in mind:

- Write a concise commit message explaining your changes.
- Mention the operating systems and versions you tested on.
- If your change affects the UI, attach a screenshot.
- Reference the issue your pull request fixes so it closes automatically on merge.
- Squash commits where possible, especially when addressing review feedback.

Before your pull request can be merged:

- The linter must pass with no warnings (`npm run lint`).
- All tests must pass.
- Your code must follow the project's conventions.
- Changes must be confirmed to work on recent versions of the supported operating systems.

## Code of conduct

This project is maintained by Balena and asks all contributors to be respectful and constructive in all interactions. If you have concerns about conduct in the project, reach out via the [Etcher forum](https://forums.balena.io/c/etcher) or the [GitHub issue tracker](https://github.com/balena-io/etcher/issues).
